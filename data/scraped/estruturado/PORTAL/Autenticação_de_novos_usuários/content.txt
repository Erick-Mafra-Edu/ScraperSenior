            Autenticação de novos usuários
            O Portal Corporativo realiza a autenticação com base na ferramenta Senior Gerenciador de Usuários. Através dela, é possível criar, editar e excluir usuários, além de definir permissões de acesso.
            Quando um novo usuário é cadastrado ou integrado ao Senior Gerenciador de Usuários e realiza a autenticação no Portal Corporativo pela primeira vez, o sistema verifica se o e-mail deste usuário atende ao formato nomedeusuario@dominio, no qual:
            
                nomedousuario: nome utilizado para identificar o usuário dentro do domínio. É composto somente por números, letras, sem espaços ou ainda caracteres especiais como cedilha, sinais de pontuação ou acentos;
                domínio: endereço que identifica um servidor na rede. No Brasil, o padrão utilizado para endereço de domínios é formado pelo nome do domínio, tipo do domínio e país.
            
            Exemplo:
                
                    Endereço: senior.com.brNome do domínio: seniorTipo do domínio: .com (Comercial)País: .br (Brasil)Domínios internacionais dispensam o elemento país no endereço.
                
            
            Caso um novo usuário tenha um e-mail fora do formato padrão ou tenha apenas o nome de usuário informado no Senior Gerenciador de Usuários, o sistema o busca o domínio cadastrado na Central de Configurações Senior, em Sistemas > Portal Corporativo > Integração de Usuários, para permitir o login no sistema. Se não houver domínio cadastrado, o sistema usará o endereço padrão instalacao.com.br para criar o e-mail.
            Exemplo:
                
                    Domínio cadastrado na Central de Configurações Senior: empresa.com.brSituação 1E-mail não informadoUsuário no SGU: joseLogin criado para o Portal: jose@empresa.com.brSituação 2E-mail com domínio incompletoUsuário no SGU: joseE-mail no SGU: jose@empresaLogin criado para o Portal: jose@empresa.com.brSituação 3E-mail com erro de digitação.Usuário no SGU: joseE-mail no SGU: jose@empresacom.brLogin criado para o Portal: jose@empresa.com.brSituação 4E-mail não informado na SGU e domínio não cadastrado na Central de ConfiguraçãoUsuário no SGU: joseLogin criado para o Portal: jose@instalacao.com.br
                
            
            Se desejar, utilize o e-mail cadastrado no Senior Gerenciador de Usuários para o recebimento de e-mails pelo Portal Corporativo, como é feito para o portlet Programação de férias.
            
                Importante
                
                Os e-mails criados automaticamente pelo sistema servem apenas para cumprir a exigência de formato para login no sistema. São e-mails nominais usados exclusivamente para autenticação, não são válidos para envio e recebimento de mensagens.
            
            Caso desejar, é possível editar o e-mail automático criado pelo sistema. O processo deve ser executado no Senior Gerenciador de Usuários, através da tela Propriedades do Usuário. Sempre será utilizado o Senior Gerenciador de Usuários como base para a autenticação, mesmo se o usuário modificar o seu cadastro no Painel de Controle do Portal Corporativo.
