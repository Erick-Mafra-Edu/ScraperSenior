            Introdução
            A interface entre os sistemas FIS e Sistema do Cliente é realizada através de requisições HTTP ou HTTPS, sob método GET ou POST, para um web service Restful enviando informações no formato JSON. 
            A URL do serviço é disponibilizada para o cliente desenvolver o programa de consumo que pode ser facilmente construído em qualquer linguagem de programação. Em cada interface é demonstrada sua respectiva URL e seu Layout.
            
                Exemplo de URL:
                
                 https://ipservidor:porta/interfacefis/saída/faturamento
                
            
            Endpoint: https://ipservidor:porta/interfacefis
            Resource: /entrada/pedido
            O cliente pode receber os seguintes códigos de retorno pelo Webservices:
            
                200 (Success): retornado quando a interface funcionar perfeitamente.
                400 (Bad Request): retornado quando alguma informação do layout ou validações não estiver correta.
                401 (Unauthorized): retornado quando as credenciais de acesso não estiverem corretas.
                404 (Not Found): retornado quando o serviço estiver fora do ar, ou não existir.
            
             500 (Internal Server Error): retornado sempre quando houver um erro interno no lado do servidor, a mensagem junto dom o código ajudará a identificar possíveis erros no layout do arquivo, por exemplo obrigatoriedade de informações.
