            Integração com SDK de Eventos CFTV
            O Ronda Senior disponibiliza uma forma para que os eventos de acesso e alarme recebidos e tratados sejam encaminhados e exibidos em aplicações de terceiros. Isso é realizado através de um SDK (Software Development Kit, ou seja, Kit de Desenvolvimento de Software ou Kit de Desenvolvimento de Aplicativos) e depende do desenvolvimento de outros fabricantes para que o recebimento dos eventos seja efetuado.
            Para habilitar essa funcionalidade, é necessário possuir:
            
                um sistema de terceiros apto para receber as informações dos eventos do sistema da Senior;
                um driver de integração (que pode ou não estar acoplado ao próprio sistema de terceiros);
                a licença de uso da Senior (campo SDK Eventos da proprietária), que determina a quantidade de dispositivos que terão seus eventos (de acesso e de alarme) encaminhados ao sistema de terceiros;
                o módulo Senior CSM Center instalado. 
            
            Configuração Senior
            Com os pré-requisitos preenchidos, configure no cadastro de dispositivos quais são os dispositivos que devem ter encaminhados os seus eventos para o sistema de terceiros. Para isso, para cada dispositivo desejado, é necessário ir até a guia Configurações e selecionar "Sim" na opção Disponibiliza Evento SDK.
            Caso o número de equipamentos com a opção acima seja maior do que o permitido pela licença de uso da Senior, os dispositivos excedentes não terão eventos enviados, ou seja, os eventos que chegarem a esses dispositivos serão ignorados.
            
                
            
            Além dos dispositivos, deve-se habilitar o envio dos eventos na CSM Center. Isso é realizado na CSM Center Config, guia Integrações. Selecione a opção Integrar com Driver de Eventos e em seguida clique no botão Configurar. O serviço da CSM Center precisa ser reiniciado para que a nova configuração seja percebida pela aplicação.
            
                
            
            Configuração do sistema de terceiros
            A informação referente a sistemas de terceiros e a forma de integração com o SDK de Eventos varia de integrador para integrador. Você deverá consultar a documentação daquele sistema para entender como os eventos recebidos são utilizados ou exibidos por ele.
            Como funciona?
            O módulo responsável por receber, tratar e encaminhar os eventos para sistemas de terceiros é a CSM Center. Durante a inicialização desse módulo, é verificada se há licença de uso para o SDK Eventos. Em caso positivo, a CSM Center determina a quantidade de equipamentos que terão seus eventos encaminhados para o sistema de terceiros. Os coletores de marcações e de alarmes devem estar rodando para que os eventos sejam encaminhados. Para cada encaminhamento, uma linha de log com as informações geradas é gravado no arquivo server.log.
