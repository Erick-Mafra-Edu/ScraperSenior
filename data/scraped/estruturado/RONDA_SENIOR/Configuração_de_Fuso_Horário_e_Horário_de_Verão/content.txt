            Configurações de Fuso 
		 Horário e Horário de Verão
            Horário de Verão é a alteração do horário de uma região, designada apenas 
 durante uma porção do ano, adiantando-se em geral uma hora no fuso horário 
 oficial local. Atualmente, o horário de verão é adotado nas regiões Sul, Sudeste e Centro-Oeste.
            A mudança de horário impacta no uso de sistemas informáticos, como é 
 o caso do Ronda Senior, razão pela qual é necessário efetuar 
 configurações para manter atualizadas as informações de servidores, bancos 
 de dados, sistema web, processos automáticos, procedimentos on-line, assim 
 como diversas integrações com hardware:
            Banco de dados
                
                    As definições de fuso horário e horário de verão repercutem nos dados 
 do banco assim como também nas informações que a aplicação utilizará para 
 efetuar suas rotinas. São várias as funcionalidades do sistema  que são influenciadas pela configuração de data e hora do 
 servidor de banco de dados, entre elas estão:
                    
                        botões de data/hora;
                        nova entrada de visitantes;
                        diversos monitoramentos;
                        emissão de relatórios;
                        execução de processos automáticos.
                    
                    Sincronização de informações
                    Acesse a Central de Configurações Senior e configure a sincronização das informações do banco de dados com a aplicação. 
 A funcionalidade está disponível para os bancos de dados Oracle e SQL Server, neste, antes de realizar a configuração do banco é necessário 
 dar a permissão CREATE FUNCTION para o usuário do CFG.
                    Configurações de GMT no servidor de banco de dados Oracle
                    Em ambientes que contêm sites com diferenças de horário (GMT ou horário de verão)  entre o banco de dados da máquina servidora e a do cliente, é necessário configurar o GMT do servidor de banco de dados Oracle. Na maioria dos servidores essa configuração está zerada.
                    O comando para configurar o banco de dados é: ALTER DATABASE SET TIME_ZONE = '-00:00'.
                    Exemplo: 
                        
                            Na região sul do Brasil, onde o GMT é -3:00, o retorno do comando 'SELECT DBTIMEZONE FROM DUAL' deverá ser -03:00 fora do período de horário de verão ou -02:00 durante o período de horário de verão.

                        
                    
                    Após realizar as parametrizações adequadas na Central de Configurações Senior, é necessário efetuar o procedimento e exclusão e recriação dos objetos da base de dados no CBDS (Central de banco de dados Senior).
                
            
            Servidor
                
                    Para que o horário de verão funcione corretamente na CSM Center e na Concentradora, o fuso horário configurado no Windows deve ser um dos disponibilizados por padrão. Não é permitido configurar um fuso horário customizado, isto é, com um nome próprio, que não seja um dos disponibilizados por padrão no Windows.
                    Exemplo:
                        
                            Forma incorreta: "(UTC-03:00) Nome do Cliente"
                            Forma correta: "(UTC-03:00) Brasília" ou "(UTC-04:00) Cuiabá"
                        
                    
                    Isso ocorre, pois Java utiliza o nome do fuso horário configurado no Windows para consultar, em sua base de dados interna de fuso horário, o GMT que deve ser aplicado. Para cada nome de fuso horário, existe um GMT sem horário de verão e um GMT com horário de verão. Por exemplo, para Brasília, se o Windows estiver fora do horário de verão, o Java utilizará o GMT -3; se o Windows estiver dentro do horário de verão, o Java utilizará o GMT -2.
                    Se o Java não encontrar o nome do fuso horário em sua base de dados, ele utilizará o GMT explícito no nome, independente de estar dentro ou fora do horário de verão, como por exemplo: "(UTC-03:00)".
                
            
            Senior Gerenciador de Usuários
                
                    Na tela de Propriedades do Usuário, na guia Fuso horário e horário de 
 verão, é possível definir se um usuário utilizará o horário de verão.
                    Caso esteja atribuído 
 na Central de Configurações Senior > Banco de Dados > Ronda Senior > Fuso Horário > Assinalamentos Globais, que o mesmo fuso horário e horário 
 de verão são aplicáveis a todos os usuários, estas opções no SGU são automaticamente 
 desabilitadas.
                
            
            Processos automáticos
                
                    Configuração de Processos Automáticos
                    Na Central de Configurações Senior, em Middleware > Web 5.0 > Processos Automáticos,  informe o fuso horário 
 do servidor onde o monitor de aplicações será executado, e se o 
 servidor utiliza horário de verão ou não.
                    Selecione essa opção de utilizar o fuso horário caso o servidor, onde o monitor de aplicações 
 é executado, use o recurso de fuso horário. Sempre que escolhida essa 
 opção é necessário definir qual será o GMT do local onde está o servidor 
 de aplicações.
                    Exemplo:
                        
                            
                                 Local 
 A: onde está alojado o servidor do Middleware = GMT -03:00 (Brasil - SC)
                                 Local 
 B: onde está alojado o servidor do banco de dados = GMT –06:00 (México)
                                 Local 
 C: onde está o usuário 1 = GMT +08:00 (China)
                                 Local 
 D: onde está o usuário 2 = GMT +05:30 (Índia)

 necessariamente a configuração deverá ser efetuada pelo Senior Gerenciador de Usuários.
                        
                    
                
            
            Locais físicos
                
                    O sistema utiliza o local físico como a 
 entidade que representa, na aplicação, um local existente na organização. 
 A configuração do fuso horário dos locais físicos é realizada no Plano 
 de Localização Física, onde são cadastrados de acordo com os  
 tipos de categorias.
                    Apenas a categoria Site permite a configuração 
 de fuso horário e de horário de verão, fazendo com que todos os locais 
 físicos cadastrados como filhos adotem automaticamente tanto o GMT 
 como as datas de início e fim do horário de verão do local pai.
                    Essas informações de GMT, início e fim do horário de verão serão utilizadas 
 na configuração de equipamentos.
