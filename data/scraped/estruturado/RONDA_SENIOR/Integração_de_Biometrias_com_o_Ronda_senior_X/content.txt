            FAQ - Integração de Biometrias com o Ronda senior X
            Os dados biométricos são compartimentados durante a transmissão e o armazenamento?
                
                    Sim. A compartimentação possibilita dividir os dados em partes menores e tratá-los de forma isolada para aumentar a segurança, sendo assim, as biometrias armazenadas passam pelo Protocolo SDK, que transforma os bytes em valores hexadecimais para interpretação com base no dicionário deste mesmo protocolo. Da parte de hardware, o fabricante poderá apoiar.
                
            
            Os dados biométricos são criptografados?
                
                    Não, é um código binário. A biometria é armazenada em banco de dados de acordo com a especificação técnica do fornecedor do dispositivo de leitura de biometria (ou tecnologia biométrica adotada).
                
            
            Para prevenção à fraude há associação de um código hash ao dado biométrico?
                
                    A responsabilidade sobre a leitura biométrica é sempre do dispositivo, não do sistema. A biometria fica armazenada no sensor do equipamento, e é apenas replicada ao Ronda senior X, mas durante a validação de acesso, a biometria é buscada no próprio sensor do equipamento, identificada, associada a uma numeração de credencial (crachá) e nesse momento o sistema Senior é acionado para checar se essa pessoa (vinculada a essa credencial) possui as permissões necessárias para ter seu acesso liberado.
                
            
            Os dados biométricos podem ser excluídos do software Ronda senior X?
                
                    Sim. Há opção de exclusão no Cadastro de Pessoas e no Painel das Credenciais.
                
            
            Após a rescisão contratual, os dados pessoais biométricos são deletados do software?
                
                    Para rescisão contratual do colaborador registrado em seu ambiente, não, a biometria permanece no Ronda senior X. No entanto,  ela será excluída de todos os equipamentos biométricos em que estava vinculada anteriormente.
                
            
            Como o algoritmo faz a identificação do titular? Verificação de um para um (autenticação) ou identificação de um para muitos?
                
                    As duas formas são aceitas pelo Ronda senior X, mas se tratando especificamente de biometria facial, apenas um para muitos.
                
            
            Quais são as categorias de dados pessoais registrados no software?
                
                    A solução já disponibiliza um mapeamento de todos os dados pessoais utilizados na ferramenta, o que agiliza o trabalho dos clientes para a realização do levantamento. É com base nesse mapeamento que as rotinas de consulta e exclusão dos dados pessoais serão executadas. O mapeamento sobre os dados pessoais e dados pessoais sensíveis é realizado de acordo com as definições da LGPD (Lei nº 13.709/2018). O cliente pode alterar o mapeamento e modificar as informações sugeridas pela Senior, assim como adicionar ou excluir campos com dados pessoais.
                
            
            Qual o período previsto de retenção dos dados ou critério para o final do tratamento?
                
                    Fica a critério do cliente, não do sistema. É possível usar a mesma opção supracitada ou sugerir uma melhoria através do Fórum de Produto.
                
            
            Existe algum procedimento de monitoramento e correção de eventuais erros no reconhecimento facial?
                
                    Essa funcionalidade fica a cargo do fabricante do equipamento facial, recebemos apenas as requisições e processamos as aceitas. A calibragem do equipamento também é do fabricante.
