            Ferramenta de Diagnóstico
            Esta ferramenta é responsável por realizar o diagnóstico do Ronda Senior, apresentando relatórios sobre o horário de verão e a carga de pessoas nos dispositivos REP.
            Importante
                Este serviço é disponibilizado em interface web para os navegadores Chrome, Mozilla Firefox e Internet Explorer  (na versão do 8 ou posterior).
            
            Para que a Ferramenta de Diagnóstico seja apresentada, é necessário assinalá-la ao realizar a instalação do sistema no Servidor.
            Consulte aqui as configurações necessárias na Central de Configurações Senior.
            Além disso, a URL para conexão JDBC deve estar configurada na Central de Configurações Senior obrigatoriamente.
            Dados de acesso
            Porta padrão do serviço: http://<servidor>:8079
            O usuário e senha para acesso à Ferramenta de Diagnóstico são os mesmo utilizados para acesso ao sistema.
            Ao acessar a Ferramenta de Diagnóstico pela primeira vez, ocorrerá uma falha na autenticação, conforme imagem abaixo:
            
                
            
            Crie uma nova senha no SGU e tente acessar a ferramenta novamente. Esse processo é necessário para atualização do método de criptografia de senhas.
            
                Observação 
                A alteração de senha é necessária para apenas um usuário do sistema.
            
            Se o erro abaixo ocorrer, solicite à Senior uma proprietária válida:
            
                
            
            Relatórios
            Horário de Verão
            Uma varredura é realizada na base para identificar se a configuração do dispositivo está correta, e se o comando de carga de horário de verão foi enviado e executado com sucesso. 
            Sempre que o comando é executado são listadas a data e a hora, além das seguintes informações:
            
                Horário atual do banco de dados;
                Site;
                Coletor;
                Dispositivo;
                Modelo;
                Fabricante do dispositivo;
                Local físico;
                Fuso horário;
                Inicio do horário de verão;
                Término do horário de verão;
                Central de gerenciamento;
                Endereço da central de gerenciamento;
                Código do comando a ser enviado;
                Comando executado com sucesso.
            
            Carga do REP
            Uma varredura é realizada na base para identificar se o colaborador tem todas as informações necessárias para ser carregado no REP e se o comando foi enviado com sucesso para o dispositivo. 
            Sempre que o comando é executado, são listadas a data e a hora, além das seguintes informações:
            Dispositivos Telemática:
            
                Nome do colaborador no equipamento;
                Número do PIS no equipamento;
                Comando de carga no equipamento;
                Código da operação de carga no equipamento;
                Número do PIS na carga do equipamento.
            
            Dispositivos SDK:
            
                Comando de carga no equipamento (REP);
                Operação no comando de carga;
                Número do crachá no comando de carga (físico);
                Tecnologia do crachá no comando de carga;
                Nome do colaborador no comando de carga;
                Número do PIS no comando de carga.
            
            Dados comuns apresentados para todos os fabricantes:
            
                Horário atual do banco de dados;
                Nome do colaborador;
                Número do PIS;
                REP On-line;
                Verifica permissão;
                Permissão dias normais;
                Permissão feriado;
                Permissão sábado;
                Permissão domingo;
                Permissão de área restrita;
                Número do crachá;
                Número físico do crachá.
