            Cockpit
            Ferramenta de acesso aos processos de negócios com opções de gerenciamento e de visualização e edição dos processos dos demais usuários. Sendo que para   estes depende das permissões que o perfil (usuário, administrador ou coordenador) possui no sistema. O Cockpit é um ambiente web, por isso para inicializá-lo é necessário primeiro efetuar o login no sistema para seja realizada uma verificação das permissões  de acesso.
            Funcionalidades:
            
                Administrar processos;
                Iniciar, acompanhar e cancelar as solicitações dos processos ;
                Tratar pendências.
            
            
                Nota
                
                 O Cockpit pode ser facilmente customizado pelo administrador quando necessário.
            
            A grade da área de visualização de dados pode ser alterada, adicionando ou removendo colunas, alterando o seu tamanho ou posição. Esta alteração é mantida mesmo que visualizado outro formulário, tela ou ainda aplicado outros filtros.
            Acesso
            O acesso ao Cockpit é realizado através da URL configurada no momento da instalação do Workflow, acrescida do sufixo "cockpit". Ao acessar, informe os dados de acesso, os mesmos utilizados no sistema Gestão de Pessoas | HCM.
            Após informar os dados, uma autenticação é realizada antes do Cockpit estar disponível para utilização. As validações realizadas são:
            
                Usuário e senha informados são válidos;
                Conta do usuário está bloqueada;
                Senha do usuário expirou e precisa ser alterada;
                Arquivo de proprietária é válido.
            
            O acesso também está disponível através de outro aplicativo web, utilizando o recurso de login Single Sign-On (SSO).
            Sessão expirada
            Por padrão, a sessão do usuário logado expira após 30 minutos de inatividade no sistema, este tempo não pode ser alterado. Ao expirar a sessão, o sistema apresenta uma mensagem informando que a sessão expirou e, em seguida, volta para a tela de login.
            
                Notas
                
                O servidor de aplicação determina se o timeout será respeitado ou não. Sendo que na tela Minhas pendências não ocorrerá timeout, pois possui atualização automática, não necessitando de interação do usuário para manter-se ativa.
            
            Iniciar processo
            Crie um novo processo. Ao pressionar este botão é exibida uma lista de todos os processos   disponíveis para executar. Este ícone padrão do processo pode ser definido no momento da criação ou customização do processo.
            Caso o  modelo do processo esteja configurado para que uma tela seja aberta, preencha os dados solicitados e confirme para que o processo seja iniciado. Se não estiver configurado para abrir, o Cockpit iniciará a execução do processo.
            
                Nota
                
                Antes de iniciar  são efetuadas as seguintes validações:
                
                    A proprietária possui os produtos necessários para iniciar o processo, caso contrário não é iniciada;
                    Verifica a quantidade de execuções de processos Workflow contratadas, se este número já tiver sido atingido, o processo não é iniciado.
                
                Essas validações são realizadas tanto para processos com e sem tela inicial. Sendo que quando tem tela inicial, as validações são realizadas após o preenchimento e confirmação dos dados na tela, já quando não tem, é realizado ao iniciar o processo.
