            Modo reparação
            O modo de reparação permite que um ambiente com sistemas Senior instalados passem por um processo de correção de alguns eventuais problemas, como por exemplo, um web service ou uma forma de acesso WindowsAccess, BrowserAccess e Web 5.0) publicada incorretamente no servidor Java EE. 
            Esse modo consiste basicamente em realizar novamente o deploy no servidor Java EE de todos os artefatos que compõem a forma de acesso ou, caso o ambiente utilize integração através de web services, todos os web services utilizados serão publicados novamente, com exceção das customizações realizadas localmente, estas sempre serão mantidas.
            Durante esse processo, todas as instâncias do sistema deve estar paradas, ou seja, nenhum usuário pode estar utilizando. Também recomendamos a realização de um backup completo, para evitar perda de dados ou arquivos personalizados.
