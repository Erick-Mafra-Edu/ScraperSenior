            Orientações para utilização dos web services
            Para evitar problemas no processamento dos web services dos sistemas Senior, é obrigatório cumprir as seguintes orientações:
            Chamada de web em processos executados via Middleware
            
            Um web service do sistema Senior poderá ser executado através de um outro tipo de processo que por sua vez é executado no Senior Middleware. Por exemplo:
            
                 Uma regra customizada é executada via Processo Agendado (que são executados via Senior Middleware). Esta regra faz a chamada de um web service padrão de um sistema Senior;
                Um web service customizado é executado em modo síncrono (via Senior Middleware). Este web service faz a chamada de um web service padrão de um sistema Senior.
            
            Nestes cenários, o web service do sistema Senior deverá ser executado também através do Senior Middleware. Ou seja, o acionamento do web service não poderá ser realizado através do modo de execução Local.
            Exemplo:
                
                    Cenário incorreto:
                    
                        Criação de um web service customizado no sistema Gestão Empresarial | ERP;
                        O web service customizado faz a chamada de um web service padrão do sistema Gestão Empresarial | ERP;
                        Na regra  é definido como “ModoExecucao” do web service padrão, o valor “1 - Modo Local”;
                        O web service customizado é acionado através de um sistema de terceiros através do Senior Middleware indicando processamento síncrono.
                    
                    Cenário correto:
                    
                        Criação de um web service customizado no sistema Gestão Empresarial | ERP;
                        O web service customizado faz a chamada de um web service padrão do sistema Gestão Empresarial | ERP;
                        Na regra  é definido como “ModoExecucao” do web service padrão, o valor “2 - Modo Síncrono”;
                        O web service customizado é acionado através de um sistema de terceiros através do Senior Middleware indicando processamento síncrono.
                    
                
            
            
                Importante
                
                Caso a orientação indicada acima não seja seguida, poderão ocorrer diversos problemas no processamento da rotina, como o retorno de "Access Violation".
                A utilização dos web service de maneira incorreta, conforme cenários descritos, não é homologada pela Senior. Portanto, caso identifique a utilização de algum cenário não homologado, é pré-requisito que seja efetuado ajuste no processo conforme estas orientações.
                A mudança para a chamada em modo síncrono também possibilitará verificar através da consulta de requisições dos web services os parâmetros de entrada e a saída da execução de cada web service do sistema Senior, facilitando análises que envolvem chamada de web services padrões dentro de web services customizados.
            
            Configuração do tipo de autenticação de um web service customizado que faz a chamada de um web service padrão
            
            Ao realizar a chamada de um web service padrão do sistema Senior a partir de um web service customizado, é importante verificar o parâmetro Autenticação da tela de cadastro da porta do serviço. Caso seja utilizada a autenticação Anônimo no web service customizado, a eventual chamada de um web service padrão na regra do customizado poderá retornar uma mensagem de erro de credenciais inválidas, como se o login no sistema não tivesse sido executado adequadamente.
            Isso ocorre, pois dependendo do web service padrão que for chamado na regra do web service customizado é obrigatória a autenticação de um usuário previamente cadastrado no sistema. A autenticação é realizada através do web service customizado e é mantida no momento da chamada ao web service padrão.
            Caso ocorra esta situação, é necessário garantir que o web service customizado esteja parametrizado para o tipo de autenticação Fazer logon.
