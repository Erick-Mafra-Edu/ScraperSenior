            Tabelas temporárias
            O recurso de tabelas temporárias existe nos bancos de dados Oracle e SQL Server e é utilizado para otimizar determinadas rotinas do banco de dados.
            Este recurso pode ser utilizado pelos sistemas Senior para que algumas rotinas tenham sua performance melhorada e está disponível apenas para algumas tabelas. O recurso pode ser habilitado e desabilitado utilizando-se o CBDS.
            
                Importante
                
                A configuração de tabelas temporárias deve ser alterada apenas sob as recomendações de um DBA que tenha previamente avaliado o banco de dados onde este recurso será utilizado.
            
            As tabelas que podem ser definidas como temporárias, na guia Detalhes possuem o campo Obs. informando que a tabela permite ser definida como temporária.Para definir uma tabela como temporária, é necessário clicar com o botão direito do mouse sobre a tabela e escolher a opção Definir a tabela como temporária. Ao ser marcada como temporária, o campo Obs. passará a informar que a tabela está definida como temporária.
            Apesar de ser marcada como temporária, a tabela não será criada como temporária no banco de dados automaticamente. Para isso, é necessário recriar a tabela, usando a tela do menu Ferramentas > Alterar Tabelas, recriando também todas as triggers e Stored Procedures do banco. E ainda recriar todas as views clicando com botão direito, na opção Views da treeview, e escolher a opção Enviar para e Executar.
            Após estes procedimentos, o sistema passará a usar as tabelas marcadas como temporárias.
