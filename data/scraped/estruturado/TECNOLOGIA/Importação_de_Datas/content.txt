                Importação de Datas
                Para variáveis de regras
                Para importar datas diretamente para variáveis da regra, pode-se proceder de duas maneiras, a primeira é selecionar o tipo Data na definição do Layout, a segunda é importar separadamente os componentes da data (dia, mês e ano) e depois uni-los na regra.
                Para campos do tipo Data
                Para importar para campos do tipo Data, deve-se a colocar o campo no Layout e informar a máscara adequada. 
                Importante
                    Campos com apenas dois algarismos no ano podem ser apresentadas como "DDMMYY" ou "DD/MM/YY". Para que estas máscaras sejam importadas corretamente, é preciso informar o ano base no modelo.
                
                Na tela Importação e Exportação de Dados (Definição), dentro da guia Definições, a caixa de seleção Considerar o ano de <ano_base> como ano base para máscaras de data com YY no ano, permite indicar para a rotina de importação, que a consistência de campos obrigatórios deve ser realizada.
                Importante
                    
                        Quando este recurso está desabilitado, o ano base usado é o padrão do sistema (normalmente 1900).
                        Os modelos que são convertidos para a versão onde este recurso está disponível têm como padrão o ano base desativado. Ou seja, os modelos vão funcionar como na versão anterior a este recurso. Caso seja necessário importar datas com apenas dois algarismos no ano com uma data base diferente, deve-se habilitar e informar o ano base no modelo.
                        O recurso de ano base fica sempre desabilitado quando o modelo está aberto em modo de compatibilidade.
