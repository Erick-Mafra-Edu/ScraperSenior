            Importador/Exportador de Artefatos em JSON e arquivo de texto
            A funcionalidade de importar e exportar artefatos permite gerenciar arquivos JSON e arquivos de texto, além de armazenamento em diretórios externos.
            Essa estrutura possibilita o versionamento dos modelos por meio de repositórios de controle de fontes, facilitando o acompanhamento de alterações e a reutilização dos modelos conforme necessário.
            Os artefatos que podem ser importados e exportados são:
            
                Modelos de relatório
                
                Regras LSP
                
                Web services
                
            
            
            Exportação de Artefatos
            Permite exportar modelos de relatórios, regras LSP e web services em formato JSON, armazenando-os no diretório configurado.
            Passo a passo para a exportação
            
                Abra a Tela de Exportação, pelo seguinte caminho Menu > Recursos > Implementações > Gerador de Importação/Exportação;A estrutura dos menus pode variar de acordo com o sistema utilizado. Após acessar Menu > Recursos > Implementações, os itens disponíveis podem ser organizados de formas diferentes, conforme o contexto do sistema;
                Visualize a lista de artefatos disponíveis para exportação:Adicione os artefatos desejados, clicando com o botão direito sobre o nome do artefato para Adicionar ou Remover da Lista de Exportação;
                Confirme o diretório de exportação onde os arquivos serão salvos:O diretório deve ser configurado na Central de Configurações Senior, a partir da chave: com.senior.<produto>.reports.artifacts_shared_dir;Se a chave não estiver configurada, será solicitado que  informe o caminho manualmente;
                Verifique os modelos adicionados no campo correspondente ao artefato (Relatórios, Regras ou Web Services);
                Clique no botão Exportar para gerar os arquivos JSON;
                Acompanhe o status em Logs de Exportação.
            
            Como os arquivos exportados são gerados
            Arquivos gerados por modelos de relatório:
            Cada modelo exportado criará uma pasta com os seguintes arquivos:
            
                General_Infos.json: Informações gerais do modelo;
                Change_History.json: Histórico de alterações;
                Base_Structure.json: Estrutura principal do relatório;
                Arquivos adicionais JSON: Representam seções específicas do relatório.
            
            Arquivos gerados por regras LSP:
            
                Cada regra LSP exportada será disponibilizada em um subdiretório chamado Regras.
                O nome da regra será o diretório correspondente à regra exportada. Dentro deste diretório haverá 2 arquivos:um arquivo JSON com definições do artefato;um arquivo TXT com o conteúdo da regra.
            
            Arquivos gerados por web services:
            
                Ao exportar um web service, será exibida uma lista dos web services editáveis criados no provedor interno, assim como também os provedores e serviços externos.
                No diretório de exportação, todos os web services serão disponibilizadas em um subdiretório chamado Web Services. Neste diretório será criada a seguinte estrutura de subdiretórios:<Nome_Provedor>/<Nome_Serviço>/<Nome_porta>Os arquivos criados dentro dessa estrutura serão:Provedor: caso seja um provedor externo, será criado um arquivo JSON de definição com as informações do provedor ao qual o serviço pertence.Serviço: caso seja um serviço customizado, será criado um arquivo JSON com as definições do serviçoPorta: será criado um arquivo JSON com as definições da porta. Caso o serviço do provedor seja interno, será criado um arquivo com a Regra do serviço.
            
            
            Importação de Artefatos
            Permite importar artefatos previamente exportados em arquivos JSON, possibilitando a criação de relatórios, regras ou web services com base na estrutura desses arquivos.
            Requisitos para a importação
            A pasta do diretório selecionada deve conter a seguinte estrutura de arquivos:
            
                General_Infos.json
                
                Change_History.json
                
                Base_Structure.json
                
                Outros arquivos JSON, representando seções do relatório.
            
            Passo a passo para a importação
            
                Abra a Tela de Importação, pelo seguinte caminho Menu > Recursos > Implementações > Gerador de Importação/Exportação;A estrutura dos menus pode variar de acordo com o sistema utilizado. Após acessar Menu > Recursos > Implementações, os itens disponíveis podem ser organizados de formas diferentes, conforme o contexto do sistema;
                Informe o diretório raiz de importação:Por padrão, o sistema utiliza a chave de configuração: com.senior.<produto>.reports.artifacts_shared_dir, que deve ser configurada na Central de Configurações Senior;Se a chave não existir, informe o diretório manualmente;
                Aguarde a listagem dos modelos de relatório disponíveis no diretório;
                Adicione os artefatos desejados:Clique com o botão direito sobre o artefato desejado;Selecione Adicionar à Importação;
                Verifique os artefatos adicionados no campo Modelos selecionados;
                Clique no botão Importar para processar os arquivos;
                Acompanhe o status no campo Logs de Importação.
            
            O que ocorre com artefatos importados com o mesmo nome
            Ao importar artefatos com o mesmo nome de artefatos já existentes, ocorre o seguinte:
            
                Modelos de relatório: caso já exista um modelo com o mesmo nome, o modelo existente será substituído pelo novo modelo que foi importado. Senão, um novo modelo será criado.
                Regras LSP: caso já exista uma regra com o mesmo nome, ela será substituída pela nova regra que foi importada. Senão, uma nova regra LSP será criada com o nome da regra exibida.
                Web services: será exibida uma lista dos web services disponíveis para importar. A importação seguirá as regras abaixo:Provedor: caso o provedor já exista, ele não será alterado. Caso não exista, será criado um novo provedor com as informações salvas no arquivo.Serviço: caso o serviço já exista, ele não será alterado. Caso não exista, será criado um novo serviço com as informações salvas no arquivo.Porta: todas as informações das portas serão substituídas pelo arquivo que está sendo importado.
