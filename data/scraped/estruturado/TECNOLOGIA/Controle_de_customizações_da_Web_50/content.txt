                Controle de customizações da Web 5.0
                É possível controlar os arquivos HTML customizados utilizados na forma de acesso Web 5.0
                Veja abaixo a explicação do funcionamento desse recurso:
                Instalação
                Durante a instalação dos aplicativos que serão acessados através da Web 5.0, o Instalador armazena um resumo de cada arquivo HTML, esses arquivos representam o estado deles no final da instalação.
                Atualização
                Durante a atualização de versão é gerado um novo resumo de cada arquivo que foi previamente instalado. Esse novo resumo  é comparado com o resumo que está armazenado, esses resumos serão comparados no momento da atualização e caso existam diferenças será apresentada uma tela ao usuário, para sobrescrever ou manter o arquivo atual. Para os arquivos customizados há uma verificação adicional que indica se eles foram modificados pela nova versão liberada pela Senior ou pelo usuário.
                Quando não houver diferenças nos resumos gerados, os arquivos serão atualizados sem nenhum tipo de aviso ou pendência de confirmação por parte do usuário.
                Configuração
                Ao alterar a URL de algum aplicativo web através da Central de configurações, será apresentada uma mensagem informando que serão feitas as seguintes operações:
                
                    Atualização dos links estáticos nos arquivos de páginas;
                    Atualização dos resumos para que os arquivos não sejam apontados como customizados na próxima atualização.
                
                É possível ignorar este procedimento pelos seguintes motivos:
                
                    A configuração está sendo ajustada para se adequar ao que está nos arquivos. Neste caso, tanto os links quanto os resumos já estão corretos;
                    Todos os arquivos são customizados, então nem perde tempo com ajuste de resumos.
