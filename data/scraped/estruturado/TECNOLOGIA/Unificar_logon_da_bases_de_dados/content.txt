            Unificar logon de bases de dados
            A unificação de logon de bases de dados é utilizada para que qualquer logon dos sistemas G5, de uma instalação, seja concentrado em uma única base de dados. Configurações relacionadas a permissões ainda são realizadas individualmente em cada base de dados. 
            Para que o recurso seja totalmente utilizado é preciso configurar alguns recursos na Central de Configurações Senior e, assim, garantir a existência do usuário nas duas bases de dados:
            
                Acesse Central de Configurações Senior > Opções de segurança > Logon via base de autenticação Senior;
                Marque as opções Permitir logon via base de autenticação Senior e Permitir logon proprietário Senior caso falhar via base de autenticação, para que assim, caso um logon na base falhe, seja realizada uma segunda tentativa no sistema proprietário utilizando as mesmas credenciais informadas;
                Selecione a Base de dados para autenticação;
                Marque a opção Sincronizar informações do usuário ao autenticar para garantir que bases de dados mantenham as informações integradas, como, por exemplo, senhas e e-mail;
                Em seguida, acesse a tela Formas de logon por tipo de acesso e configure as diferentes formas de acesso para cada para cada tipo 
 de solução disponível instalados atualmente, que estejam utilizando o logon da base Senior;
                 Após configurar as formas de login, acesse a tela Importação de usuário para obter os usuários da base do sistema secundário, por exemplo,  obter o do Gestão Empresarial | ERP para importá-los para a base de autenticação do Gestão de Pessoas | HCM. Na tela, marque somente o sistema que terá que enviar os usuários;Exemplo:
                Para garantir que alterações de usuários realizadas no SGU dos sistemas sejam atualizadas nos demais sistemas presentes na instalação, é necessário, na tela Replicação de usuários, habilitar as bases que fazer e aceitam replicação;ImportanteNessa configuração, o sistema selecionado para exportar e replicar os usuários  deve possuir os web services publicados e funcionando corretamente, pois o Senior Gerenciador de Usuários utiliza os web services deste sistema para obter os usuários.
                Em seguida, acesse a tela Autenticação de usuários e certifique que os usuários novos existam nas bases do sistema, através da seleção da opção Permitir adicionar novo usuários a um grupo automaticamente. Se desejar, habilite a configuração para gerar esses novos usuários em uma base secundária em seu primeiro logon através da adição destes usuários em um grupo pré-existente na nessa base secundária através do campo Adicionar novos usuários ao seguinte grupo automaticamente;
                Por fim, após essas parametrizações na Central de Configurações Senior, acesse o Senior Gerenciador de Usuários, em Arquivo > Importar Usuários... e importe os usuários da base secundária para a base principal. Clique em Importar de base Senior via web service para visualizar os usuários a serem importados e os que já existem e serão atualizados. Nesse caso, importe somente os novos usuários, que contam com +, pois caso os demais sejam atualizados, esses usuários perderão as permissões e grupos da base corrente.
            
            
                Importante
                
                Caso entre as bases existam objetos com o mesmo nome (seja usuário ou grupo), por exemplo,  importar um usuário com nome Senior em uma base com um grupo de nome Senior pré existente, a importação do usuário não irá ocorrer até que este conflito esteja resolvido alterando o nome do objeto da base destino ou base origem e assim, removendo o conflito.
