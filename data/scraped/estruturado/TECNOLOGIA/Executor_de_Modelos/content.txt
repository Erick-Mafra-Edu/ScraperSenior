            Robô
            Passagem de Parâmetro
            
            O sistema suporta passagem de parâmetro 
 na sua inicialização. Os seguintes parâmetros são suportados:
            
                 -fn:file_name: 
	  Sendo file_name o caminho completo do arquivo de modelo, ex:C:\Temp\Robô\DG00001.DGN.
                Abre o modelo passado como parâmetro.
                -execute: 
	 Executa o modelo passado como parâmetro. Esse parâmetro deve ser usado 
	 junto com o do item 1;
                
                    Se o modelo possuir variáveis de entrada, posiciona na tela 
		 para o usuário informar as variáveis de entrada
                
                -splash: 
	 Não exibe a tela de abertura na inicialização do aplicativo.
            
            
                
                    Observação
                    
                    Os exemplos citados variam de acordo com cada versão (exemplo: C:\Temp\Robô\DG00001.DGN).
                
            
            Inicialização
            Ao executar o arquivo do aplicativo (Diagnostico.exe), abrirá a tela 
 para seleção do arquivo .CFG referente ao sistema que será considerado 
 para o diagnóstico. Ele é homologado para ser executado apenas nos servidores, não em estações.
            
                
            
            Basta selecionar o arquivo e clicar no botão Ok.
            É possível adicionar um arquivo .cfg na lista para seleção, clicando 
 no botão Novo.
            
                
            
            Para criar um arquivo, clique no botão [...] e o aplicativo irá abrir 
 no diretório do arquivo padrão. Mas se for necessário, é possível buscar 
 outros arquivos em diretórios diferentes.
            
                
            
            Ao selecionar o arquivo do diretório, clique no botão Abrir 
 o nome do arquivo será preenchido na tela de Edição 
 da lista do CFG, adicione uma descrição e clique no botão 
 Ok
            Também é possível editar o nome ou o diretório de um arquivo já existente, 
 como na criação de um arquivo novo.
            
                
            
            O botão Opções permite que as seguintes configurações sejam 
 feitas:
            
                
            
            
                Sempre pedir para o usuário escolher 
	 qual arquivo de configuração será aberto: Com essa opção marcada, 
	 sempre que o arquivo for executado, abrirá a tela para selecionar 
	 o .CFG a ser usado.
                Abrir o seguinte arquivo de configuração: Se 
	 essa opção for marcada, é preciso informar qual o arquivo .CFG será 
	 considerado padrão, e ao inicializar o aplicativo, a tela para seleção 
	 do arquivo .CFG não será apresentada.
                Abrir o arquivo de configuração 
	: O aplicativo carrega os arquivos .CFG existentes 
	 no diretório em que o executável está, e os apresenta na lista. Para 
	 que a opção acima funcione, um arquivo deve ser selecionado neste 
	 campo.
                Arquivo de configuração padrão: 
	 Se o EditCFG possuir um arquivo padrão definido, o aplicativo irá 
	 buscar esse mesmo arquivo para permitir configurar ou não um arquivo 
	 .cfg padrão para o Robô. O nome do arquivo será preenchido no campo 
	 Arquivo de configuração padrão:.
            
            A seguir, é preciso realizar o logon com um usuário administrador do 
 sistema.
            
                
            
            Durante a inicialização, é apresentada uma tela contendo:
            
                
                    Logo e site 
		 oficial da Senior;
                    Nome do produto;
                    Versão do produto;
                    Status da inicialização;
                
            
            Conforme a imagem abaixo:
            
                
            
            Tela Principal
            Após a inicialização, o aplicativo se apresenta dividido em três menus 
 principais e a área de edição do modelo, como mostra a imagem abaixo:
            
                Arquivo;
                
                Opções;
                
                Ajuda;
                
            
            
                
            
            No canto inferior esquerdo, é mostrado o nome do usuário administrador 
 logado na ferramenta. Nesse exemplo, é o usuário Senior.
            Para utilizar o Executor de Modelos, 
 é recomendado que as configurações 
 já tenham sido feitas. Depois disso, é preciso apenas executar o atalho 
 para a ferramenta.
