            Restore da base de dados
            O restore da base de dados é indicado para a montagem de um ambiente de homologação, que anteriormente estava em produção. 
            Porém, após realizar este processo, todas as informações que estavam no ambiente de produção são levadas também para o ambiente de homologação, o que pode ocasionar alguns impasses, como: 
            
                envio de e-mails para usuários do ambiente de produção. Após o restore, todas as informações dos usuários do ambiente de produção constam no ambiente de homologação;
                processos disponíveis para executar sendo publicados no ambiente de produção e com tarefas que executam web services.
            
            Portanto, é indicado que após o restore da base de dados, que está com todas as informações do ambiente de produção, seja feito alguns ajustes:
            
                acesse o Senior Gerenciador de Usuários e remova os e-mails dos usuários que constam no ambiente de homologação;
                republique todos os processos no ambiente de homologação, afim de evitar a execução indevida de um web service.
            
            
                Observação
                
                Caso possua também a base de dados do Gestão de Pessoas | HCM, essas ações devem ser realizadas também nessa base.
