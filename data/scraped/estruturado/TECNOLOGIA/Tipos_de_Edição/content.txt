            Tipos de Edição
            Tipos de Caracteres Entre os Separadores: As Informações especiais de máscara estão situadas entre os separadores '#' no inicio da edição. Entre eles temos:
            
                < Indica que a edição deve ser processada da direita para a esquerda. Ex. #<# 999.999
                > Indica que a edição deve ser processada da esquerda para a direita. Ex. #># 9.9999
                b Brancos se For Tudo Zerado Sem mostrar a máscara.
                n Brancos se For Tudo Zerado mostrando a máscara vazia.
                - Sinal de menos caso este campo possa receber valor negativo, se ele receber um valor negativo a for editado, um erro de transformação de máscara será gerado em tempo de execução.
                M Indica que o valor apesar de ser do tipo Dinheiro a formatação do campo será o mesmo do tipo Numérico.
                V Oculta a vírgula de um valor. Exemplo: #V2#zzzzz9,99 - 1534,67 = 153467.
                [] Indica o inicio e fim de um campo do tipo especial. Entre estes caracteres deve ser inserido o nome do especial desejado.
                W Oculta caracteres especiais de um campo cadastro. Exemplo: #W#AAAAAA - "Edição" = "Edicao"
            
            Alfanumérico - Cadeia de Caracteres: Permite A,L,U,Z,9 e qualquer outro tipo de caractere especial.
            
                Exemplo:
                    
                        
                            (z99)999-99999
                            AAA-UUU-LLL
                            ULLLLLLLLLLLLLLLL
                        
                    
                
            
            Numérico: Permite Z,9 e os seguintes caracteres especiais. .,/ ,: ,(, ), [, ].
            
                Exemplo:
                    
                        
                            (z99)999-99999
                            #-<b# 9999999 permite digitar o valor negativo, caso o valor seja zero, nada aparecerá.
                            zzz.zzz.zz9
                        
                    
                
            
            Como padrão este tipo de campo será editado da direita para a esquerda.
            Data: Permite D,M,Y com caracteres especiais / ou -.
            
                Exemplo:
                    
                        
                            DD/MM/YY data simples
                            DD-MM-YYYY data com ano completo
                            MM/YY para competência simples
                            MM/YYYY para competência com ano completo.
                            #b#DD/MM/YY se a data for zerada brancos serão mostrados.
                            #n#DD/MM/YY se a data for zerada será mostrado o seguinte / / .
                        
                    
                
            
            Dinheiro: Permite z,9 com caracteres especiais '/' - '.' - ','.
            
                Exemplo:
                    
                        
                            #-# ZZZ.ZZZ.ZZ9,99 aceita valor negativo
                            99999.9999,99
                            99999999 neste caso a edição para este tipo de dados será processada como se fosse para um tipo numérico.
                        
                    
                
            
            
                
                    Legenda:
                
                
                    
                        9 - NuméricoZ - Numérico Sem Considerar os Zero a esquerda.A - Caractere AlfanuméricoU - Caractere MaiúsculoL - Caractere MinúsculoD - DiaM - MêsY - Anom - Minutoh - Hora.
