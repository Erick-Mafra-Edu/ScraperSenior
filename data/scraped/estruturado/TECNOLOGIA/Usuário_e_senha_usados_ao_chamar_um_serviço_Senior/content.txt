                Usuário e senha usados ao chamar um serviço Senior
                Há duas formas de autenticação para web services:
                Anônimo: Não 
 faz o logon no sistema limitando o acesso à base de dados e, dependendo 
 do sistema, às rotinas.
                Com logon: 
 Quando um web service que está configurado para fazer logon é chamado 
 à partir de um sistema da Senior, o usuário e a senha autenticados no 
 sistema são usados para a autenticação desse web service. A forma de logon 
 aplicada será conforme a configuração das Formas 
 de Logon aos sistemas por Tipo de Acesso feitas no SGU.
                Requisitos para funcionamento do acesso com logon:
                
                    O Java deve estar acessível através do path do Windows;
                    O aplicativo "crypt.jar" deve estar no diretório da aplicação;
                    Os servidores devem ter o horário sincronizado devido ao recurso 
	 expirar o token em 1 hora.
                
                O acesso com 
 o logon funciona da seguinte maneira:
                
                    Há um serviço cadastrado no provedor interno chamado "WSCalcula";
                    Nesse serviço está cadastrada a porta "somar" que está 
	 configurada para fazer logon;
                    Está logado no sistema o usuário "senior" com a senha 
	 "123";
                    Executa-se essa porta de forma síncrona; 
	mesmo comportamento para execuções assíncronas e agendadas.
                    O serviço faz logon em uma instância do sistema no servidor com 
	 o usuário "senior" e a senha "123".
