            Central de Segurança
            Permite configurar as várias opções de segurança para grupos e usuários.
            Para selecionar um Grupo ou Usuário, utilize o botão busca (dentro do campo Usuário/Grupo), será exibida a tela Selecionar usuários/grupos. Para visualizar todos os usuários, utilize o botão Mostrar Usuários. 
            Outra opção para a seleção de usuários é 
 o assinalamento Exibir painel de usuários, que exibe uma 
 lista de usuários logo acima da caixa de configuração de permissões. Este 
 painel de usuários contém todos os usuários e grupos que tenham alguma 
 permissão configurada para o item selecionado na lista à esquerda. Ao selecionar algum usuário, 
 a caixa de configuração de permissões será atualizada com as permissões 
 do usuário selecionado, assim como a caixa de texto Usuário/Grupo.
            
                Nota
                
                O painel de usuários está disponível apenas para as guias Tabelas 
 e campos, Telas, Geradores, Regras 
 e Configurações. Ao habilitar o painel em alguma outra guia, 
 ele será exibido vazio e desabilitado.
            
            Tabelas 
 e campos
                
                    Nesta guia, encontram-se as permissões para tabelas e campos. Ao alterar o valor de uma destas configurações, o botão Aplicar ficará habilitado.
                    A finalidade desta tela é a mesma que a do CBDS, ou seja, definir qual processo o usuário poderá fazer ao nível de banco de dados em relação ao campo da tabela.
                    Quando definido que o usuário/grupo tem apenas a opção "Consultar" liberada, ao tentar processar o registro para ser feito a inserção ou atualização da informação no banco de dados, será retornado uma mensagem de "Acesso negado a tabela" no momento em que o registro é processado, impedindo que o valor seja gravado. Porém, quando se tratar de uma tela gerada, ao restringir a permissão para um determinado usuário, o campo ficará inativo para alteração.
                    Esse comportamento é independente de qual tela/campo foi realizado o bloqueio na Central de Segurança, pois se trata de uma configuração relacionada a tecnologia, e não em relação à rotina da tela em si.
                    
                        Importante
                        
                        No caso do usuário ser um administrador, ou estar dentro de um grupo administrador, essas configurações da Central de Segurança não têm efeito, visto que, usuário administrador têm acesso total ao sistema, independentemente de outras parametrizações.
                        Para o sistema Gestão Empresarial | ERP, verifique mais detalhes sobre a configuração dos campos na Central de Segurança do Gestão Empresarial | ERP.
                    
                
            
            Telas
                
                    Representam a estrutura de menu da tela 
 principal para acessar as Telas da Aplicação. Se for selecionado um item 
 que não é referenciado em uma tela do sistema, por exemplo, Arquivo 
 e Sair, não serão apresentadas as opções Alterar, 
 Excluir e Inserir.
                    Quando existir vários módulos (guia Módulos), 
 o sistema irá verificar a permissão do usuário no módulo e se não tiver 
 permissão de alteração não exibirá as telas daquele módulo.
                
            
            Geradores
                
                    
Apresenta a Lista de Relatórios, Consultas, Importação, Exportação e Cubo 
 Decisão.
                    Semelhante às telas, não havendo permissão para Alterar Permissões, será possível apenas 
 visualizar as configurações de permissão do usuário selecionado e alterar/visualizar permissão de um item de nível 
  4, pois os anteriores representam categorias. As opções de abrir e executar  o modelo devem ser utilizadas para realizar o bloqueio de visualização.
                
            
            Regras
                
                    
Apresenta as Regras de cada módulo, definidas pela aplicação de 
 acordo com os diretórios específicos. 
                
            
            Serviços multicamada
                
                    Apresenta as configurações de permissionamento dos web services.
                    Administrar serviços
                        Indica se pode manipular os web services através da tela Console de Administração de Serviços.
                    Executar serviços síncronos
                        Permite definir se será possível executar os web services síncronos.
                    Executar serviços assíncronos
                        Indica se pode executar os web services assíncronos.
                    Executar serviços agendados
                        Permite definir se será possível executar os web services agendados.
                    Executar serviços agendados recorrentes
                        Indica se podem ser executados os web services agendados recorrentes.
                    Altera permissões de serviços
                        Indica se o usuário pode configurar as permissões para outros usuários/grupos.
                
            
            Workflow
                
                    Apresenta a lista de processos do Workflow que estão publicados.
                    Alterar data/prazo limite
                        Permite alterar a data/prazo limite do processo via cockpit.
                    Alterar permissões
                        
                        Possibilita alterar permissões do processo
                    Alterar proprietário
                        Permite alterar o proprietário do processo via cockpit.
                    Cancelar execução
                        Permite cancelar a execução do processo via cockpit e WS.
                    Executar
                        
                        Possibilita iniciar os processos via:
                    
                        Cockpit;
                        WS e JMS;
                         API SeniorTools.
                    
                    Exibir histórico
                        
                        Permite definir como um determinado usuário irá proceder em relação a um histórico iniciado pelo administrador. As opções disponíveis são:
                    
                        Nunca: não permite o usuário consultar o histórico de execução do processo;
                        Quando iniciar o processo: permite que o usuário consulte o histórico do processo quando foi ele quem iniciou a execução;
                        Sempre: permite que o usuário consulte o histórico do processo mesmo quando não foi ele quem iniciou a execução.
                    
                    Coordenar
                        Permite que o usuário coordene os fluxos, podendo alterar os prazos, responsáveis e demais dados do fluxo.
        
                    
                        Nota
                        
                        Por padrão, quando publicado o processo pela primeira vez, o processo tem acesso total a para todos os usuários.
                    
                
            
            Perspectivas
                
                    Esta guia tem a função principal de conceder permissões a usuários e grupos de usuários, para acessar menus e funcionalidades no Gestão do Ponto. 
                    As permissões aplicadas nesta guia, relacionadas ao Analytics, só terão as configurações aplicadas ao menu "Meu analytics" e "Analytics da minha equipe". No que diz respeito ao acesso no analytics do Gestão do Ponto Nova Interface, não há tratamento separado de permissões para os cards. Se um colaborador ou gestor tiver acesso ao analytics, terá acesso aos cards completos sem distinção.
