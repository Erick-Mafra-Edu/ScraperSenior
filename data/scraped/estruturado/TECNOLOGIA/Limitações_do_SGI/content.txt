            Limitações do SGI
            Campo com máscara especial
            Os campos com máscaras especiais são definidos em tempo de execução, ou seja, quando o sistema abre, muda esta máscara conforme condições internas. No CBDS (Central de banco de dados Senior) não há estas condições, por isto o campo é considerado de acordo com o tipo de campo informado.
            Campos Chave
            Todo formulário X-Window necessita de uma tabela-base e seus respectivos
		campos chaves. Os formulários Web não necessitam, 
mas se uma tabela-base for informada, ela obrigatoriamente necessita de campos chaves. 
            Para que um campo possa ser informado como campo chave, 
		alguns requisitos devem ser obedecidos:
            
                O campo não pode ser do tipo Lookup;
                O campo deve possuir um campo físico correspondente;
                O campo deve estar na tabela principal ou em uma tabela da direita ligada em 1-para-1 com a tabela
			principal. Mais informações em Ligações Simples e Ligações Customizadas.
            
            Tabelas
            Duas tabelas informadas na Interface e relacionadas 
		entre si com uma ligação 1-para-1, 1-para (1-opcional), ou 1-para-n, não 
		é permitem criar no mesmo formulário, uma grade de ambas as tabelas. Se uma tabela da Interface for utilizada no campo Tabela 
 nas propriedades de uma página, a mesma tabela não pode ser usada em uma  grade. O SGI não permite que a mesma tabela do SGI seja usada em dois lugares 
		do formulário. Por "lugar do formulário", entende-se:
            
                Campo Tabela-base;
                Campo Tabela nas propriedades de uma página;
                Campo Tabela-base nas propriedades de uma grade.
            
            Limitações na Web
            Em tabela-base de uma grade não é possivel efetuar a carga de dados dos campos de tabela. Essa funcionalidade apenas é suportada nas telas SGI X-Window.  

            Não é possível realizar a atribuição de valores para campos de tabela, devido ao conteúdo das telas SGI Web não ser dinâmico
como em telas SGI X-Window. 
            A variável CodOpe não se atualiza de maneira semelhante ao X-Window, é necessário realizar a utilização dos botões padrões para que o valor da variável receba os valores diferentes do status "Iniciando".
            Tradução
            Quando existir uma personalização de menus de SGI, para alterar a descrição de algum menu é necessário acessar a tela Personalização de menus. Caso a descrição seja alterada através do SGI ou pela tradução do menu da tela, ela não será gravada na personalização, ocasionando erros na visualização dos menus. Pois com a personalização de menu é mantida a descrição do menu configurada na personalização.
            Campos lógicos
            Ao adicionar campos lógicos (campos que não existem na tabela física) em uma tabela e efetuar uma ligação deste campo com uma outra tabela, apenas para criar o campo com o B2 resulta em um comando SQL inválido, pois o sistema não tem como resolver essa ligação. Este comando inválido pode não retornar valores corretamente, dependendo da versão do banco de dados. Para solucionar essa situação, o recomendável é criar na tabela um campo de usuário e alterar o campo lógico para corresponder ao campo de usuário criado, desta forma, o comando SQL gerado é válido e todos os bancos de dados conseguem executar corretamente.
            Palavras reservadas na nomenclatura dos campos
            Os campos criados em tabelas via SGI não poderão conter uma palavra reservada de banco de dados em sua nomenclatura, pois ocasionarão erro em tela ao executar a sua consulta ao executar o formulário que a utilizar. Exemplos de palavras que não deverão ser utilizadas: SELECT, AS, FROM, ORDER, BY, ASC, DESC, UPDATE, DELETE, entre outras.
