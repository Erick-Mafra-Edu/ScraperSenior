                Importação de Dados via Tabela do Banco de Dados
                Existe um recurso na ferramenta Importador, que permite a importação de dados seja feita a partir de uma tabela conhecida do TBS do banco de dados. Este recurso pode ser utilizado para realizar uma melhor integração de sistemas.
                Glossário
                Sistema Externo 
                Sistema de outra empresa que não seja do grupo Senior que deseja integrar seus dados.
                Sistema Senior: 
                Um dos sistemas do grupo Senior que permitirá que dados de um Sistema Externo sejam integrados na sua base de dados.
                Tabela de Importação: 
                Local onde serão inseridos os dados do Sistema Externo e de onde o Sistema Senior deverá ler para realizar o processo de importação.
                Disponibilização de dados pelo Sistema Externo
                O Sistema Externo que deseja integrar dados com o Sistema Senior, deverá disponibilizar seus dados e uma Tabela de Importação, que esteja definida dentro da base de dados usada pelo Sistema Senior. Esta tabela deverá estar definida no TBS do Sistema Senior e pode ser uma tabela de usuário, criada a partir do aplicativo CBDS da Senior Sistemas.
                Os dados que estarão contidos nesta tabela, deverão estar no formato padrão do Importador de Arquivos Texto, ou seja, se o layout for fixo, as posições, os tamanhos e as máscaras deverão ser respeitados. Se o layout for variável, os valores deverão possuir um separador, este que é definido no modelo de importação de dados (geralmente ponto-e-vírgula).
                Tabela de Importação
                A tabela de importação deve representar um arquivo de texto, pois durante a importação, é criado um arquivo de texto com os campos TEXTn da tabela, ou seja, os campos TEXTn de cada linha da tabela representam uma linha no arquivo utilizado na importação.
                A chave primária da tabela é definida pelo usuário, sugere-se utilizar como chave primária os campos TEXTn, pois não pode haver linhas duplicadas na tabela.
                STATUS (ou USU_STATUS) 
                Campo obrigatório de tipo number(1), que armazenará o estado de cada registro.
                
                    
                        Valor
                        Estado
                    
                    
                        0
                         Indica que a linha ainda precisa ser processada.
                    
                    
                        1
                        Indica que a linha foi importada com sucesso.
                    
                    
                        2
                        Indica que a linha foi importada com erro. Se a linha deve ser importada novamente, é necessário mudar o estado para 0 (zero).
                    
                    
                        9
                         Indica que a linha está sendo processada.
                    
                
                TEXTn (ou USU_TEXTn)
                
                Campo obrigatório de tipo varchar, sendo que o n no nome, deve ser um número sequencial (0, 1, 2, 3...).
                SeqID (ou USU_SeqID) 
                Campo opcional de tipo number(9), que se preciso, armazenará a sequência na qual os registros devem ser importados, sendo crescente a ordem de importação.
                Exemplo da  Tabela de Importação
                
                    
                    
                    
                    
                    
                    
                        
                            Campos
                            STATUS
                            SEQID (opcional)
                            TEXT0
                            TEXT1 (Tamanho: 200)
                        
                        
                            Registro 1
                            0
                            0
                            0;100;CAM001;10;01/01/2001;NOME COM MUITOS SOBRENOMES QUE ULTRAPASSA DUZENTOS CARACTERES SE A LINHA NÃO CABE NO TEXT0, O EXCEDENTE DEVE SER PREENCHIDO NO TEXT1 XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX;31/1
                            2/2001
                        
                        
                            Registro 2
                            0
                            1
                            0;101;CAM001;200;01/01/2001;NOME SOBRENOME;31/12/2001
                            (NULO)
                        
                        
                            Registro 3
                            0
                            2
                            1;456;CAM001;200;01/01/2001;NOME SOBRENOME;31/12/2001 
                            (NULO)
                        
                    
                
                Nota
                    
                        Os dados da tabela de exemplo foram dispostos em layout variável (conceito do Importador de Arquivos Texto).
                        Caso o conteúdo do registro não caiba no TEXT0, o conteúdo excedente deve ser preenchido nos campos TEXTn seguintes. Por exemplo, o campo TEXT1 deve ser preenchido apenas quando o campo TEXT0 estiver preenchido, ocupando o tamanho total da coluna.
                    
                
                Importação de dados no Sistema Senior
                O Sistema Senior deverá providenciar um arquivo temporário, seguidamente deverá executar uma rotina que obterá os dados da Tabela de Importação, então estes dados serão armazenados no arquivo temporário determinado anteriormente. Por fim, o Sistema Senior executará a rotina de importação de dados utilizando o arquivo temporário que contém os dados da Tabela de Importação.
                
                    Nota
                    
                    O Sistema Senior poderá apagar o arquivo temporário gerado anteriormente (sujeito a critérios específicos do sistema).
