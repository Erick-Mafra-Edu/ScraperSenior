            Comandos SQL
            O sistema permite que sejam adicionadas e excluídas ações, cada ação 
 pode ter um comando SQL. 
            
                
            
             
            Aba 
 Geral:
            Nome: 
 Nesse campo deve ser informado o nome 
 do comando SQL que será utilizado no executor, o nome deve ser único, 
 e auxiliar na identificação dos demais comandos registrados.
            Descrição: 
 A descrição deve conter a especificação 
 do que o comando faz, justificativas ou outros detalhes relevantes.
            Comandos 
 SQL: Neste 
 campo, deve ser inserido um comando do tipo select para retornar dados 
 da base. Como nenhum dado pode ser alterado, o sistema deve permitir apenas 
 comandos do tipo select. Caso algum comando que altere a base (como ‘delete’, 
 ‘alter’, ‘insert’, ‘update’, etc) for informado, o sistema deve gerar 
 uma exceção com a mensagem "O comando SQL pode ter apenas instruções 
 do tipo select." e a operação deve ser abortada.
            O 
 sistema também valida o comando ‘select’, exibindo a mensagem de erro 
 "Comando SQL inválido. Erro:msg_erro.", onde msg_erro é a mensagem 
 de erro retornada pelo banco de dados caso este comando esteja incorreto. 
 A validação do comando SQL será feita somente se o item 'Utilizar SQL 
 Senior 2' estiver marcado.
            O 
 sistema permite que o usuário informe variáveis de entrada no comando 
 SQL.
            O 
 sistema valida o comando SQL. O sistema deve 
 exibir a mensagem de erro "Comando SQL inválido. Erro:msg_erro.", 
 onde msg_erro é a mensagem de erro retornada pelo baco de dados caso o 
 comando SQL esteja incorreto. A validação do comando SQL será feita somente 
 se o item 'Utilizar 
 SQL Senior 2' estiver marcado.
            Plano 
 de ajustes (tempos por ocorrência): Link 
 para a documentação.
            Excluir 
 linhas sem ocorrência: Link 
 para a documentação.
            Utilizar 
 SQL Senior 2: Habilita a utilização 
 de SQL Senior 2.
