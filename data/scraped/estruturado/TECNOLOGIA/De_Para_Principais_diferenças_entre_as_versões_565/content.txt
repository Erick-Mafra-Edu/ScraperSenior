            Tela de escolha de CFG para acessar os sistemas
            A tela de seleção de CFG foi ajustada para poder selecionar o ambiente desejado para acessar um sistema. Nos aplicativos: SGU, CBDS e VisLog, além do ambiente pode-se selecionar também a base de dados utilizada para efetuar a conexão.
            Tela escolha de CFG padrão
            A tela de seleção de CFG padrão foi ajustada para poder selecionar o ambiente e base de dados padrão para acessar o sistema.
            Recursos deprecados
            
                Logon com usuário do sistema operacional. Contínua disponível apenas para quem estiver usando o recurso. Ao efetuar uma conversão é solicitado se deseja trocar de forma de acesso. A forma de logon sugerida é NTLM.
                Visualização de relatórios com ActiveViewer na Web 5.0. Contínua disponível apenas para quem estiver usando o recurso. Ao efetuar uma conversão é solicitado se deseja trocar para PDF.
                O uso do parâmetro BASE nas aplicações Web 5.0 deve ser substituído pela utilização de um “ambiente” para cada BASE, ou resolvido através da infra estrutura de rede e configuração dos servidores do cliente.
            
            Exemplo: 
            
            Parâmetro BASE nas aplicações Web 5.0
            
                Cenário antigo (mesma instalação):RUBI com BASE=producaoRUBI com BASE=desenvolvimentoHTMS’s e/ou CFG’s diferentes
                Cenário novo (mesma instalação):RUBI com AMBIENTE=producaoRUBI com AMBIENTE=desenvolvimentoHTMS’s e/ou AMBIENTES diferentes
