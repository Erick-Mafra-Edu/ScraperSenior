            Dados alfanuméricos em Oracle
            Para tratamento dos dados alfanuméricos em banco de dados Oracle, os parâmetros relevantes são:
            Configuração de código de página
            Os dados alfanuméricos apresentados ao usuário e armazenados na memória pelos sistemas Senior usam o código de página Windows-1252. Com isso, os parâmetros de database devem ser configurados com os valores que tiverem a maior compatibilidade possível com este código de página e o menor esforço na tradução dos caracteres entre a aplicação e o banco de dados.
            
                NLS_CHARACTERSET:  os valores homologados pela Senior para este parâmetro são WE8MSWIN1252, WE8ISO8859P1 e WE8ISO8859P15;
                NLS_NCHAR_CHARACTERSET: o valor deste parâmetro não tem influência nos sistemas da Senior, já que, nativamente, não são usados tipos de dados que dependem deste parâmetro, como NVARCHAR e NCLOB. O valor recomendado é AL16UTF16.
            
            Configuração de ordenação
            Para funcionar de forma adequada, os sistemas Senior necessitam usar a mesma regra de comparação e ordenação, que no banco de dados Oracle são definidas pelos parâmetros de sessão nsl_comp e nsl_sort. 
            Os sistemas Senior configuram o parâmetro nsl_comp = 'LINGUISTIC'. O parâmetro nsl_sort depende da linguagens cadastradas em Linguagens de Ordenação, no CBDS.  Na ausência de linguagens cadastradas e habilitadas, os sistemas apresentam os dados alfanuméricos usando as regras compatíveis com a linguagem 'AMERICAN' que corresponde ao comportamento padrão do Oracle (nls_sort = BINARY). Caso contrário, os dados alfanuméricos serão apresentados usando as regras compatíveis com as da linguagem definida pelo software cliente do banco. O valor é nsl_sort = 'WEST_EUROPEAN' para 'BRAZILIAN PORTUGUESE', por exemplo.
            
                Observação
                
                Os sistemas Senior não realizam verificação para garantir que a linguagem configurada no software cliente do banco de dados está entre as linguagens configuradas no cadastro. Se o sistema for usado nesta situação, podem ocorrer problemas de performance. Para saber mais sobre esta situação, pesquise por Oracle Globalization Support Guide na sua versão do banco de dados Oracle.
                O código de página ISO-8859-1¹ não é totalmente compatível com o código de página Windows-1252². Portanto haverá caracteres que não possuem representação no código de página ISO-8859-1¹.
