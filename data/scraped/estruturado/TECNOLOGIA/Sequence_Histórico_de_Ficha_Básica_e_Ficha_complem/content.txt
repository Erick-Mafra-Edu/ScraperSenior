            Sequences Histórico de Ficha Básica e Ficha Complementar
             
            Histórico da Ficha Básica e Complementar (Oracle)
            Para que o recurso de histórico da ficha básica e complementar funcione adequadamente em bases Oracle é necessário que existam as seguintes  sequences* no banco:
            
                R038HFB_SEQ;
                R038HFC_SEQ.
            
            
                Nota
                
                Sequence é um objeto do banco de dados, a partir do qual vários usuários podem gerar números inteiros exclusivos. É possível utilizar essas sequences para gerar, automaticamente, valores de chave primária. Para mais informações acesse o documento Create Sequence, do banco de dados Oracle.
            
            Esses objetos são criados automaticamente durante a atualização do sistema, porém , em caso de exclusão acidental, é necessário recriá-los manualmente a partir dos seguintes passos:
            
                Verificar se as sequences estão presentes no Banco:Execute o seguinte comando, utilizando o CBDS ou outr ferramenta de banco: SELECT * FROM USER_SEQUENCES WHERE SEQUENCE_NAME='R038HFB_SEQ' OR SEQUENCE_NAME='R038HFC_SEQ';
            
            
                Observação
                
                Se entre os objetos listados estiverem as sequences R038HFB_SEQ e R038HFC_SEQ, não será necessário recriá-las.
            
            
                Busque o valor inicial de cada sequence:Execute os seguintes comandos para encontrar o valor inicial de cada sequence:R038HFB_SEQ: SELECT NVL(MAX(SEQHIS),0)+1 FROM R038HFB;R038HFC_SEQ: SELECT NVL(MAX(SEQHIS),0)+1 FROM R038HFC;
            
            
                Recrie as sequences:Execute os seguintes comandos para recriar as sequences, informando os valores iniciais verificados anteriormente:R038HFB_SEQ: CREATE SEQUENCE R038HFB_SEQ START WITH {ValorInicial} INCREMENT BY 1;R038HFC_SEQ: CREATE SEQUENCE R038HFC_SEQ START WITH {ValorInicial} INCREMENT BY 1;
            
            
                Observação
                
                Ao final verifique se todas as triggers e procedures estão válidas no banco de dados.
