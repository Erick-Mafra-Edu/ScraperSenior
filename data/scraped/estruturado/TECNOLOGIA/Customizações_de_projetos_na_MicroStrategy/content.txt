            Customizações de projetos na MicroStrategy
            Novo objeto
            Para criar um novo objeto (relatório, cubo, dashboard) customizado, deve-se criar um novo projeto  e nele toda a estrutura necessária a estes objetos. Esse procedimento é necessário, pois, ao atualizar o sistema, as customizações podem ser perdidas.
            Objeto nativo
            Para utilizar algum objeto nativo, deve-se replicar manualmente estes objetos para o novo projeto criado. Para simplificar o trabalho, pode ser realizada a importação de um pacote nativo, criando assim uma estrutura inicial de projeto e posteriormente realizar as customizações necessárias. Todos os objetos de projetos novos utilizados devem ser renomeados para “USU_” + o nome do objeto, para evitar que em futuras atualizações ocorra alguma perda de informação.
            Imagens e plugins 
            Para utilizar imagens ou plugins customizados, e que sejam mantidos entre atualizações do sistema, deve ser criado um diretório chamado customizado:
            Para Imagens
                Para utilizar imagens customizadas, e que sejam mantidos entre atualizações do sistema, é necessário criar um diretório chamado customizado:
            
                "<Dominio GlassFish>\applications\MicroStrategy\imagens"
                "<Dominio GlassFish>\applications\MicroStrategyMobile\images\" (se possuir Mobile).
            
            Para plugins
                Para utilizar plugins customizados, e que sejam mantidos entre atualizações do sistema, os diretório não podem começar com "bi-" e devem ser criados em:
            
                "<Dominio GlassFish>\applications\MicroStrategy\plugins\"
                "<Dominio GlassFish>\applications\MicroStrategyMobile\plugins\" (se possuir Mobile).
            
            Mapas
            Para manter mapas customizados é necessário:
            
                O nome da pasta de mapas ser "customizado";
                A pasta estar dentro de <Dominio GlassFish>\applications\MicroStrategy\swf\kml.
            
            Conexão MicroStrategy
            Ao conectar em um ambiente customizado, com versão inferior ao que está sendo utilizado, uma mensagem de erro é exibida informando que isto pode causar um risco de corromper a estrutura do projeto MicroStrategy. Caso for uma versão superior, há possibilidade ocorrer um erro de conexão.
            
                Importante
                
                Para visualizar informações específicas da MicroStrategy, acesse o Ajuda da ferramenta.
