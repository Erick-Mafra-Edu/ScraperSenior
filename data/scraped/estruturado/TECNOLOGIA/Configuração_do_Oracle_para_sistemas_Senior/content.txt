            Configuração do Oracle para os sistemas Senior
            Considerando alguns parâmetros do banco de dados Oracle e dos Sistemas 
 Senior formatou-se uma lista de configurações para apresentar o melhor 
 resultado de performance. Todas essas configurações se fazem necessária, 
 qualquer configuração diferente da indicada não vai garantir os resultados 
 esperados, sendo necessário uma análise individual ao ambiente em uso 
 em busca desse melhor resultado. As configurações indicadas são:
            Modo de otimização
                Essa configuração exige que o banco de dados esteja com as estatísticas 
 sempre atualizadas. 
 Com função de orientar o otimizador do Oracle a utilizar 
 melhor os índices para os Aplicativos Senior, pode ser acrescentado na 
 chave de banco de dados "Comandos SQL de inicialização", 
 os seguintes comandos:
            ALTER SESSION SET OPTIMIZER_INDEX_COST_ADJ 
 = 30;
            
            ALTER SESSION SET OPTIMIZER_INDEX_CACHING = 70;
            
            Número de linhas trazidas por vez
                Na Central de Configurações Senior, na chave de banco de dados indique a opção Avançado e informe o valor 20.
            Configuração de performance de telas
                
                Na tela de Configuração de performance da Tela configure na 
 guia Dicas para comandos Select:
            
                Selecione Nenhum (Conforme o banco de dados);
                Marque a opção "Colocar dica para usar melhor índice".
            
            Na guia Buscar descrições de ligações, selecione Com vários select's (um para cada campo).
            Configuração de performance em relatório
                
                Na tela Configuração de performance, em  Gerador de Relatórios, 
 Diversos > Configuração de Performance, informar:
            
                RowSet Size = 200;
                Em "Enviar dicas para os comandos selects", selecione Primeiro 
 Registro(First Rows);
                Marcar a opção Colocar dica para usar melhor índice.
            
            Recursos de banco de dados
                
                Ter exclusivo para a base Senior:
            
                tablespace de dados;
                tablespace de índice;
                tablespace de classificação(sort);
            
            
                Nota
                
                Recomenda-se criar todos os tablespaces da base Senior para gerenciamento 
 local com EXTENT MANAGEMENT LOCAL, ou seja, o gerenciamento 
 e alocação de área dos segmentos passa a ser do Oracle aplicando regras 
 próprias conforme universo de cada tabela. Esta configuração é mais eficiente, 
 oferece maior performance do que o gerenciado por dicionário e reduz a 
 necessidade de manutenção do DBA. 
            
            Tabelas em Cache
                Tendo recursos disponíveis é recomendado a inclusão de algumas tabelas 
 "chaves" dos sistemas em cache. A lista das tabelas recomendadas 
 pelos sistemas podem ser conseguidas junto a seus manuais.
            A definição de quais tabelas colocar em cache deve vir de uma análise da base do cliente, onde identificará as tabelas que são mais utilizadas e trarão um ganho de desempenho.
            É fundamental definir um processo para manutenção das tabelas/índices 
 da base de dados Senior com atenção especial para a fragmentação dos blocos 
 e número de extends.
