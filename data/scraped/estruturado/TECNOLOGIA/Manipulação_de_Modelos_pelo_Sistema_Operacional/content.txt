            Manipulação de Modelos pelo Sistema Operacional
            Os geradores de informação (gerador de relatórios, gerador de consultas, gerador de cubos), editor de regras e o importador/exportador de arquivos texto, usam o sistema de arquivos do sistema operacional para armazenar os modelos criados a partir deles. Apesar de estarem disponíveis como arquivos, cópias feitas diretamente pelo sistema operacional não são recomendadas.
            Além das informações que os usuários fornecem durante a confecção, os modelos usam informações do sistema durante a sua geração. Isto faz com que os modelos sejam adequados para o sistema que os gerou. Dentre estas informações, está o controle de acesso dos modelos.
            O controle de acesso é baseado em um identificador gerado pelo sistema e que é gravado no modelo e no banco de dados. No banco de dados também são gravadas as permissões de cada usuário no modelo que contém o mesmo identificador. A cada novo modelo que é criado, um novo identificador é gerado. Quando um modelo é excluído usando o sistema, o identificador também é excluído do banco de dados.
            Portanto, a cópia dos arquivos usando o sistema operacional pode fazer com que o sistema se comporte de maneira incorreta. No caso das permissões de acesso, copiar um modelo para um novo arquivo usando nomes que sejam reconhecidos pelo sistema, faz com que o sistema use o mesmo identificador para dois modelos diferentes. Mudando as permissões de acesso para um modelo, também são alteradas as permissões do outro.
            Além das informações sobre as permissões de acesso, as versões internas e externas do sistema são usadas nos modelos. As versões externas são aquelas registradas no TBS (Versão de Ambiente, Versão de Inovação, Versão da Base de Dados e Versão de Compilação). As versões internas compreendem a versão das variáveis de sistema e de funções de programador. Estas versões são usadas para verificar se os modelos precisam ou não de atualização. Quando estes números não são idênticos, as ferramentas disparam uma atualização do modelo que permite que ele se adeque ao sistema que o está abrindo.
            Copiar os modelos pelo sistema operacional pode ocasionar erro na leitura dos modelos. Se o modelo for copiado de uma versão mais nova do sistema para uma versão mais velha, o modelo pode não ser lido corretamente pela ferramenta. E a diferença entre as versões nem precisa ser tão grande: as vezes uma versão de compilação diferente já pode ocasionar problemas.
            Outra situação que pode acontecer é o modelo rodar em sistemas diferentes que tem a mesma versão.  Nestes casos as ferramentas não tem como detectar as mudanças e não realizam nenhum tipo de conversão. Neste caso podem ocorrer problemas de acesso a campos do TBS. Isto acontece porque a ordem das tabelas e dos campos é diferente e alguns modelos usam esta ordem para acessar tais recursos. E como não ocorre nenhum tipo de conversão, o acesso fica incorreto ocasionando problemas durante a execução.
            Os problemas são resolvidos se os modelos forem copiados pelo sistema usando os comandos próprios para isto. O comando mais comum é Salvar Como ou Copiar. Este permite que o modelo seja salvo num novo arquivo só que levando em consideração as informações dependentes do sistema. Ou seja, para o caso das permissões de acesso, um novo identificador é gerado quando um modelo é copiado desta maneira.
