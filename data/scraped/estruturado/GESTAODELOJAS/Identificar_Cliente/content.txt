                Identificar o cliente no PDV
                 Conceito    
                A identificação do cliente é algo muito importante para manter seu cadastro atualizado.
                Utilização     
                É utilizado para identificar o cliente no processo de venda. O cliente deverá ser obrigatoriamente identificado nas vendas com intermediação de serviços de garantia estendida e seguro parcela protegida, bem como, para serviços de fretes e montagens. 
                Processos
                O cliente pode ser identificado após a abertura do cupom e antes de sua finalização.
                Na Venda 
                Para identificar o cliente em uma venda, é necessário ter iniciado seu atendimento através do botão Venda.O cliente poderá ser identificado antes ou depois de informar os produtos da venda. Para isso, siga os passos:
                
                    Selecione a opção Identifica;
                    Selecione a opção Cliente;
                    Informe o CPF ou nome parcial do cliente que está efetuando a compra ou utilize o botão pesquisar;
                
                
                    Notas 
                    
                        Informando o CPF do cliente, o sistema encontrará o cadastro, caso ele possua, e o incluirá na venda.
                        Informando o nome parcial, o sistema exibirá todos os registros com o mesmo filtro informado na pesquisa. O usuário terá que selecionar o cliente desejado, informando o código correspondente.
                        Caso o cliente não possua cadastro, o usuário deverá cadastrá-lo no próprio PDV.
                        A Situação SPC de um cliente é exibida com borda em vermelho se é igual a "3 - Negativado própria empresa" ou "4 - Negativado terceiro". Para exibição dessa informação de SPC junto ao cliente, é necessário que o parâmetro dinâmico Entidade.ExibeInformacoesSPC esteja definido como "Sim" no cadastro da filial do Gestão Empresarial | ERP.
                    
                
                 
                Via identificação de consumidor final
                
                Identificar um consumidor final significa identificar dados de um cliente sem que o mesmo seja cadastrado no sistema. Esta opção pode ser utilizada por exemplo, para atender a Nota Paulista, Nota Gaúcha etc.
                Os dados de: Nome, CPF/CNPJ e endereço informados são impressos no cupom fiscal. Através da configuração Entidade.InformacoesConsumidorFinal é possível definir quais as informações que serão são solicitadas.
                Se for informado um CPF/CNPJ existente na base do Retaguarda este registro será atribuído como cliente na venda. Se forem informados todos os dados necessários para cadastrar um cliente no Retaguarda, um cadastro de cliente será gerado com os dados informados para o consumidor.
                Ao informar um CEP, será consistido e sugerido os demais campos do endereço. Caso seja informado um CEP inexistente, o uma solicitação de confirmação será realizada para continuar ou não com a operação. Ao continuar, o cadastro desse CEP terá de ser realizado posteriormente.”
                
                    Esse recurso é aplicável apenas para PDV PAF.
                
                Na recarga
                Para identificar o cliente em uma recarga, siga os mesmos passos citados na identificação da venda.
