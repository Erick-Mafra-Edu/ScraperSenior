            Gerador de relatórios
            Conceito
            Esse recurso possibilita associar a menus do sistema relatórios personalizados, no formato JRXML (criados na ferramenta Jaspersoft Studio 6.5.1).
            Para mais detalhes sobre o funcionamento da ferramenta Jaspersoft Studio, acesse a documentação aqui.
            Associando um relatório a um menu do Retaguarda
            Para definir um relatório para um menu do sistema:
            
                Acesse Diversos > Gerador de relatórios;
                Clique no botão Incluir;
                Informe o Código do relatório e também sua Descrição;
                Selecione onde o relatório será mostrado em Localização no menu.
                No campo Localização do arquivo, informe o local onde o arquivo JRXML se encontra (observe o item Criando o arquivo de relatório para saber as melhores práticas que devem ser seguidas), ou então utilize o ícone da lupa procurar.
                Clique em Gravar.
                Acesse o menu informado no passo 4 para confirmar a criação do relatório.
            
            Veja na prática:
            
                
            
            Criando o arquivo de relatório
            Os parâmetros de entrada para o relatório criado no Jaspersoft Studio 6.3.1 para o Retaguarda devem seguir as seguintes recomendações:
            
                Ao criar o parâmetro deve ser informada a sua descrição, pois é ela que é apresentada ao usuário no sistema Retaguarda, para solicitar o parâmetro:Um parâmetro criado assim no Jaspersoft Studio 6.3.1...Gerará um campo assim no relatório do Retaguarda...
                Os seguintes tipos de classes de parâmetros são suportados pelo Retaguarda:Uma classe criada assim no Jaspersoft Studio 6.3.1...Gerará um campo assim no Retaguarda...java.lang.BooleanO sistema apresentará um checkbox para o usuário: Para que venha marcado por padrão, deve ser informado “true” no Default Value Expression nas propriedades do parâmetro:java.lang.Doublejava.lang.Floatjava.lang.Integerjava.lang.Longjava.lang.Shortjava.lang.Stringjava.math.BigDecimaljava.sql.Datejava.sql.Timejava.sql.Timestampjava.util.Date
            
            
                Para que um combo seja apresentado na tela de entrada de parâmetros:O nome do parâmetro deve começar com a palavra “combo”;O tipo de classe do parâmetro deve ser  java.lang.String;O Default Value Expression deve conter as opções do combo separadas por ponto e vírgula ( ; );Cada opção deve ter o seu identificador e sua descrição separados por um traço ( - ) - exemplo: “1-maçã; 2-uva”

            Configurando relatórios compartilhados em rede
            O sistema está preparado para encontrar o relatório compartilhado na rede da seguinte forma:
            
                Linux buscando relatório em diretório compartilhado em Linux (o caminho informado deve conter o prefixo do samba (smb) no seu início);
                Windows buscando relatório em diretório compartilhado em Windows;
                Windows buscando relatório em diretório compartilhado em Linux.
            
            
                Observação
                
                Linux buscando relatório em diretório compartilhado em Windows não está implementado.
