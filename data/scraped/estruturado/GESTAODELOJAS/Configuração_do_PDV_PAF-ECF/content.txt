            Configuração do PDV PAF-ECF
            Efetue as configurações conforme requisitos do PAF-ECF e programação dos Meios de pagamento, Alíquotas, Totalizadores não fiscais e Relatórios Gerenciais quando necessário e vinculá-los ao ECF instalado.
            
                Importante
                
                O ECF já deverá possuir todas as configurações ao ser lacrado na interventora, porém suas configurações ficam disponíveis para futuras inclusões de Meios de Pagamento, alíquotas, totalizadores e relatórios gerenciais.
                Por decisão estratégica da Senior, o software de PDV do Gestão de Lojas não será homologado no PAF-ECF para sistemas operacionais Linux. Sendo assim, a partir do vencimento do certificado atual (março/2020), o software não estará mais disponível para a versão Linux. Esta restrição se aplica apenas ao estado de Santa Catarina, no qual, é o único que exige a emissão do PAF para atendimento da legislação. Para os demais estados que possuem a emissão de NFC-e/S@T, o software estará disponível e recebendo as devidas atualizações.
                Para clientes com lojas em Santa Catarina que possuem o software de PDV instalado em sistema operacional Linux, será necessário migrar para o sistema operacional Windows. A condição do sistema operacional se restringe somente ao software PDV. Os servidores de loja e terminais de venda poderão continuar com o ambiente Linux. Um futuro credenciamento com o sistema operacional Linux não está descartado, porém, deverá ser alinhado com a equipe de produto do sistema de Gestão de Lojas.
            
            Pré-requisitos 
            Para configuração correta do ECF é necessário:
            
                solicitar o arquivo de autorização de ECF para Senior (nome do arquivo: ARQNS.SER);
                o CNPJ da filial no ECF deve ser igual ao CNPJ da filial no banco de dados;
                o ECF deve estar autorizado para uso na filial, no arquivo ARQNS.SER;
                o 	ECF deve estar autorizado para uso pelo Fisco (lacrado).
            
            Autorizar e reconhecer o ECF conectado 
            
                Para configurar a impressora fiscal, é necessário que o arquivo ARQNS.SER estar dentro da instalação do PDV e o ECF estar conectado à máquina;NotaCaso esteja utilizando uma ECF da Bematech, verifique se ela está com a configuração correta: no arquivo .ini (BemaFI32.ini ou BemaFI64.ini), caso a ECF seja o modelo MP-4000 TH FI, o item ProtocoloUnico deverá ser = "0". Caso sua impressora seja do modelo MP-4200 TH FI II, o item ProtocoloUnico deverá ser = "1".
                Execute o  PdvConf.exe, localizado dentro da raiz da instalação do PDV. Em seguida será apresentada a mensagem "Deseja utilizar uma impressora fiscal nesta estação?". Clique Sim para configurar;
			
                Neste momento, o sistema fará a identificação e o reconhecimento da ECF conectada automaticamente:
			
            
            
                Após o reconhecimento do ECF, será apresentada a mensagem de aviso "Por favor entre novamente no programa de configuração do PAF-ECF". Clique em OK e execute o PDVConf novamente.
			ImportanteCaso o ECF não seja reconhecido e autorizado, será apresentada a mensagem "ECF não autorizado" e todo o processo terá que ser refeito, e as configurações aplicadas no cadastro de empresa, filial e PDV deverão ser verificadas.
                Após executar o PDVConf novamente, será apresentada a tela para liberar as funções do ECF. Informe usuário e senha do sistema de PDV que possua permissão para configurar ECF. Esse usuário é cadastrado no Gestão Empresarial | ERP ou no Retaguarda.

            Ao abrir a tela do PDVConf, utilize primeiramente a guia de configuração Impressora Fiscal e em seguida a guia Programação para cadastrar novas funções para o ECF, caso seja necessário.

            Totalizador não fiscal
                
                    Os totalizadores não fiscais são todos os documentos impressos no ECF que não são comprovantes de vendas. O totalizador não fiscal é utilizado para impressão de qualquer tipo de recebimento de valores no caixa, não relacionados à venda de produtos/itens (onde são emitidos os cupons fiscais).
                    Todos os tipos de comprovantes não fiscais deverão ser programados nesta opção. Siga a sequência numérica dos totalizadores já cadastrados.

            Clique em Emitir Leitura X para a emissão da leitura X na impressora fiscal. Geralmente utilizada após a programação efetuada para verificar as opções cadastradas.                 O ajuste para ativar o horário de verão só poderá ser feito após a emissão de uma Redução Z. Portanto, deverá ser ajustado no final do dia após fechamento de caixa. Já o ajuste para desativar o horário de verão só poderá ser feito uma hora após a emissão de uma Redução Z.
