            Formas de autenticação suportadas na Web 5.0
            A Web 5.0permite as seguintes formas de autenticação:
            
                Logon nativo: o usuário deve informar seu usuário e senha na tela de login padrão da WEB 5.0.
                Autenticação integrada do Windows (NTLM): para definir essa opção é necessário seguir os passos da configuração.Essa autenticação tem algumas características:O login é automático assim que é acessada a página inicial do Sapiens WEB 5.0Não há redirecionamento para página de login proprietário caso a autenticação NTLM falhe (Exemplo: usuário não pertencente ao domínio, não importado, inativo, etc). Desta forma, não é possível fazer login com usuário nativo ou com credenciais digitadas manualmente, e não há como autenticar fora do domínio da rede do servidor NTLM. Não existe a função de logoff na WEB 5.0.Caso a autenticação integrada do Windows esteja ativada para WEB 5.0, as ações SID utilizarão as credenciais do usuário da máquina que está chamando a ação, utilizando a autenticação integrada. As ações não vão considerar o usuário e senha enviados na requisição.O agendador não utiliza autenticação integrada do Windows, continuando necessário o cadastro de usuário e senha no processo agendado.Os navegadores Firefox e Safari não suportam autenticação integrada do Windows.
