            Detalhamento técnico Shovel
            Estes são os passos necessários para configurar a solução de integração do TMS Senior com a Senior X para que possa ser usada pela primeira vez: 
            Definições do Integrador Shovel
            Como funciona o Shovel?
                
                    Shovel fornece a integração do ambiente On-Premises cliente com a plataforma SeniorX através do RabbitMQ, para que serviços possam ser executados e trafegados através de um canal seguro pela nuvem.
                    
                        
                    
                
            
            Requisitos para Instalação do Shovel
                
                    
                        Conhecimento sobre o Sistema Operacional Linux e seus comandos;
                        Conhecimento sobre a aplicação Docker;
                        Detalhamento técnico dos Pré Requisitos para instalação de Ambiente Híbridos(On-Premises x Cloud), Clique aqui.
                        Servidor físico ou Máquina Virtual (VM) com sistema Linux Ubuntu Server, versão mais recente estável (LTS) com a seguinte configuração:Processador com mínimo de 2 núcleos;2Gb de memória RAM;64Gb de espaço no diretório raiz ( / ), com possibilidade de expansão.
                    
                    Importante! O servidor deve ser acessível através da rede do cliente, e com acesso à Internet
                    
                        Nota: A plataforma está homologada em Ubuntu Server, não recomendamos outra distribuição.
                    
                    
                        Duas interfaces de rede IPV4, sendo:Primeira interface para conexão à Internet, com as portas 9090, 4369, 5671, 5672 e 15672 liberadas;Segunda interface para conexão à rede interna, com uma das seguintes portas liberadas:1433 – Quando utilizado MSSQL Server (entre 2008 R2 e 2016);1521 – Quando utilizado Oracle.
                    
                    
                        
                            Instalação da ferramenta Docker:sudo apt-get update / sudo apt-get install docker.io
                            
                            Configuração da interface de rede para o Docker:sudo docker network create --driver bridge my_network
                            
                            RabbitMQ: sudo docker run --network=my_network  -d --name="rabbitmq" --hostname=rabbitmq -e RABBITMQ_COOKIE="cookie" -e RABBITMQ_USERNAME="admin" --expose=4369 -p 15672:15672 -p 5671:5671 -p 5672:5672 seniorsistemas/rabbitmq:3.6.5
                            
                        
                    
                
            
            Importante! Esse processo é extremamente sensível, caso seja feita configuração em outro usuário da Platform pode ocorrer erros, desta forma sugerimos para que o processo da configuração de Shovel seja realizado pela Senior TI ou Infra.
